# Chubbs UI Cleanup Plan (v5.52+)

Drafted 2026-05-09 during Michael's break, after the v5.45 nav-restructure
routine (scheduled 2026-05-05) didn't deliver a PR. This is the fresh proposal
to land in lieu of that.

## Current state — 7 top-level tabs

```
[Me/Setup] [Leaderboard] [My Card] [Scramble] [Stableford] [📅 Season] [🏆 Playoffs]
```

Tabs come from `renderTabs()` at `index.html:5313`. "My Card" is conditionally
hidden based on `_myCardShouldShow()` (event-state aware).

Issues:
- 7 tabs wraps awkwardly (3 + 3 + 1 with empty space next to Playoffs)
- "Me" and "My Card" both player-centric — confusing dual labels for similar-
  sounding things
- 📅 / 🏆 emojis are visual noise
- Playoffs tab visible year-round but only useful 2 months/year (May–June)
- Scramble + Stableford both visible even when only one is being played that day

## Already shipped in v5.52 (low-risk first cut)

- Hide Playoffs tab for non-admin when no `season.playoffs.seeds` are locked
- Drop emojis from Season + Playoffs labels
- Bouncing logic so a hidden tab can't be the active state

Net effect: most of the year, non-admin sees 5 tabs (Me, Leaderboard, Scramble,
Stableford, Season). During playoff window with seeds locked: 6 tabs (adds
Playoffs). During an active event with My Card eligible: 6 or 7.

## Proposed Phase 2 — consolidate to 4 top-level tabs

Maps closer to actual user intent:

```
[Round]   [Leaderboard]   [Standings]   [Me/Setup]
```

Each "group" tab opens to a sub-segmented pill control showing the relevant
sub-views.

### Round tab
Sub-segments (only the relevant ones show based on event state):
- **Stableford** (default during Sunday Chubbs)
- **Scramble** (default during Saturday)
- **My Card** (personal scoring panel — when eligible)
- **3-Putt Poker** (side game, rarely active)

When no event is loaded, this tab shows a "Load event" CTA.

### Leaderboard tab — unchanged
Live scores, Final Two intermission card, sponsorship tile.

### Standings tab — combines Season + Playoffs
Sub-segments:
- **Season** (default)
- **Playoffs** (only when seeds locked)
- **Honeypot Publish** (admin only)

### Me/Setup tab — unchanged
Player profile / settings / admin Setup (label changes by mode).

## Visual treatment for sub-segments

Reusable `seg-pill` component (a CSS+JS pattern). Single horizontal row of pills
inside a card at the top of each grouped tab. Active pill gets gold underline,
inactive pills are muted text. Tap to switch.

```html
<div class="seg-pill">
  <button class="seg active">Stableford</button>
  <button class="seg">Scramble</button>
  <button class="seg">My Card</button>
</div>
```

## Migration order

| Phase | Scope | Effort | Risk |
|---|---|---|---|
| **0 — shipped** | Hide Playoffs when no seeds; drop emojis | done in v5.52 | low |
| **1** | Build seg-pill component + hoist Round group | ~3-4 hrs | medium — touches every score-view caller |
| **2** | Hoist Standings group (Season + Playoffs sub-pills) | ~2 hrs | low |
| **3** | Polish Me/Setup; rename "Setup" → "Admin" for consistency | ~1 hr | trivial |

Each phase is shippable independently. Phase 1 is the biggest payoff (drops
top-level tab count from 5-6 to 4) but also the riskiest because Score view
has lots of internal state and renderers (`renderDay2()`, `renderDay1()`,
`renderMyCard()`).

## Out of scope (intentionally)

- "Concede hole/match" buttons (separate matchplay UX work)
- Designated scorer mode (separate adoption-focused work)
- Onboarding tutorial / first-launch toast (training material work)

These are queued in CLAUDE.md / session notes and don't depend on the nav
restructure.

## Risk register

| Risk | Mitigation |
|---|---|
| Hidden tabs break deep links / bookmarks | The `setTab('playoffs')` call sites still work — just the tab button is hidden, not the section. Manual URL nav still functions. |
| Test users land on a tab that disappeared | Bouncing logic in `renderTabs()` already handles `state.ui.tab` becoming invalid by routing to a sensible fallback. |
| Admin-only flows hidden from non-admin look broken when admin toggles off | Already handled — masterMode flips re-render which re-runs renderTabs. |
| Customer (Terry / brain trust) can't find Playoffs preview | Keep visible for masterMode regardless of seed state. |

## Validation

After each phase, smoke test:
1. Fresh browser (no localStorage) — does the launcher show?
2. Load a past event bundle — do all relevant tabs appear?
3. Toggle masterMode — does the tab list update without page reload?
4. Inject playoff seeds via console — does Playoffs appear immediately?
5. Resize window to phone width (375px) — do all tabs fit without overflow?
