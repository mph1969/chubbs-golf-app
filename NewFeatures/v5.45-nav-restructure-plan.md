## Background

The bottom nav currently renders up to **7 tabs** (Setup, Leaderboard, My Card, Scramble, Stableford, 📅 Season, 🏆 Playoffs) depending on whether `_myCardShouldShow()` returns true. The `.tabs` grid at **line 51** is hardcoded to `repeat(5,1fr)` and wraps to `repeat(3,1fr)` at 480 px (line 374). Apple HIG and Material cap at 5; the current layout is already breaking the grid intent.

The restructure reduces top-level tabs to **4** and groups game-day content under a single "Round" tab with sub-pills.

## 1. Final Nav Tree

```
Setup
Round
  ├── Stableford      (§6.1 scoring — current #day2 content)
  ├── Scramble        (team gross — current #day1 content)
  ├── My Card         (personal real-time panel — current #mycard content; date-gated)
  └── 3-Putt Poker    (deal, summary, stamp log — extracted from Leaderboard)
Leaderboard           (unchanged — honeypot gate · Final Two card · live results)
Season
  ├── Standings       (current #season content)
  ├── Playoffs        (current #playoffs content)
  └── Honeypot        (admin-only publish controls; hidden from non-admin)
```

**Top-level tab labels (exact):** `Setup` · `Round` · `Leaderboard` · `📅 Season`

**Sub-pill labels:**
- Round: `Stableford` · `Scramble` · `My Card` · `3-Putt Poker`
- Season: `Standings` · `Playoffs` · `Honeypot`

`My Card` pill is suppressed when `_myCardShouldShow()` returns false (line 6118).  
`3-Putt Poker` pill is suppressed when `state.options.threePuttPoker` is false (line 900).  
`Honeypot` pill is suppressed when `!honeypotIsAdmin()` (line 6883).

### `renderRound()` (new function, ~60 lines)

```js
function renderRound(){
  const host = document.getElementById('round');
  if(!host) return;
  const showMyCard = _myCardShouldShow();
  const showPoker  = !!state.options.threePuttPoker;
  // Sub-tab order mirrors scrambleBeforeStableford() ordering logic (line 4971)
  const first  = scrambleBeforeStableford() ? 'scramble' : 'stableford';
  const second = first === 'scramble' ? 'stableford' : 'scramble';
  const subTabs = [
    [first,    first === 'scramble' ? 'Scramble' : 'Stableford'],
    [second,   second === 'scramble' ? 'Scramble' : 'Stableford'],
    ...(showMyCard  ? [['mycard', 'My Card']]       : []),
    ...(showPoker   ? [['poker',  '3-Putt Poker']]  : []),
  ];
  const active = state.ui.roundTab || first;
  const pills = `<div class="sub-tab-nav">${
    subTabs.map(([id,lbl]) =>
      `<button class="sub-tab-btn ${active===id?'active':''}" onclick="setRoundTab('${id}')">${lbl}</button>`
    ).join('')
  }</div>`;
  let content = '';
  if(active === 'stableford') content = renderDay2Html();
  else if(active === 'scramble') content = renderDay1Html();
  else if(active === 'mycard')   content = renderMyCardHtml();
  else if(active === 'poker')    content = renderPokerStandaloneHtml();
  host.innerHTML = pills + content;
}
```

`renderDay1Html()`, `renderDay2Html()`, `renderMyCardHtml()` are the existing render bodies extracted as HTML-returning variants. `renderPokerStandaloneHtml()` wraps the existing `pokerSection` block (lines 7306–7336) extracted from `renderLeaderboard()`.

### `renderSeasonTabbed()` (new function, ~30 lines)

```js
function renderSeasonTabbed(){
  const host = document.getElementById('season');
  if(!host) return;
  const showHoneypot = honeypotIsAdmin();
  const subTabs = [
    ['standings', 'Standings'],
    ['playoffs',  'Playoffs'],
    ...(showHoneypot ? [['honeypot', '🐊 Honeypot']] : []),
  ];
  const active = state.ui.seasonTab || 'standings';
  const pills = `<div class="sub-tab-nav">${
    subTabs.map(([id,lbl]) =>
      `<button class="sub-tab-btn ${active===id?'active':''}" onclick="setSeasonTab('${id}')">${lbl}</button>`
    ).join('')
  }</div>`;
  let content = '';
  if(active === 'standings') content = renderSeasonHtml();
  else if(active === 'playoffs') content = renderPlayoffsHtml();
  else if(active === 'honeypot') content = renderHoneypotControlsHtml();
  host.innerHTML = pills + content;
}
```

`renderHoneypotControlsHtml()` returns the admin-controls block currently inside `renderHoneypotAdminBar()` (line 6977) plus the honeypot card from Setup tab (lines 5618–5625) — merged into one panel. The `renderHoneypotAdminBar()` call in `renderLeaderboard()` at **line 7343** is removed; the inline admin bar on Leaderboard is no longer needed once controls live in Season/Honeypot.
